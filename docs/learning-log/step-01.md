## 질문 로그

### 질문1
**Q.** `Main.kt`를 `com` 폴더 하위에 만들어야 하나, `ll` 폴더 하위에 만들어야 하나?

**A.** `ll` 폴더 하위. `src/main/kotlin/com/ll` 경로 자체가 패키지 `com.ll`을 의미하므로, 커리큘럼의 `com.ll.Main` 규칙에 맞춰 `ll` 폴더 안에 생성.

### 질문2
**Q.** IntelliJ에서 파일 생성 후 코드 전체가 빨간 줄로 뜨며 인식되지 않는 이유는?

**A.** IntelliJ가 "Kotlin Class"로 파일을 생성해 3번째 줄이 `class Main {`으로 되어 있었음. 클래스 몸통 안에는 `println`, `while` 같은 실행 코드를 바로 넣을 수 없어 문법 오류 발생.

**트러블슈팅**
- 증상: `class Main { println(...) ... }` 전체에 빨간 밑줄, 컴파일 에러
- 원인: `fun main()` 대신 `class Main` 으로 파일이 생성됨 (클래스 바디에 실행문을 직접 못 둠)
- 해결: `class Main {` → `fun main() {` 로 한 줄만 수정. 코틀린은 클래스 없이 top-level 함수를 파일에 바로 선언 가능(자바와의 차이점).
