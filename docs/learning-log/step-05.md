## 요구사항 요약

`목록` 명령을 입력하면 등록된 명언을 번호 역순(최신순)으로 출력한다.

```
== 명언 앱 ==
명령) 등록
명언 : 현재를 사랑하라.
작가 : 작자미상
1번 명언이 등록되었습니다.
명령) 등록
명언 : 과거에 집착하지 마라.
작가 : 작자미상
2번 명언이 등록되었습니다.
명령) 목록
번호 / 작가 / 명언
----------------------
2 / 작자미상 / 과거에 집착하지 마라.
1 / 작자미상 / 현재를 사랑하라.
명령) 종료
```

## 질문 로그

### 질문1
**Q.** `.reversed()` 메소드는 내부적으로 어떻게 동작하는가?

**A.** 원본 리스트를 복사(`toMutableList()`)한 뒤 그 복사본을 제자리에서 뒤집어(`reverse()`) 반환한다. 즉 원본 `wiseSayings`는 절대 바뀌지 않고, 뒤집힌 새 리스트만 반환된다. 자바의 `Collections.reverse(list)`가 원본을 직접 뒤집는 것과 대조됨 — 코틀린은 `-ed`로 끝나는 함수(`reversed`, `sorted` 등)가 대체로 원본 불변 + 새 컬렉션 반환 패턴을 따름.

### 질문2
**Q.** `.reversed()`를 왜 쓰는가? (동작 원리 말고 필요한 이유)

**A.** 두 가지 이유. (1) 요구사항 자체가 "목록은 최신순으로 출력"이라 화면 표시 순서를 뒤집어야 함. (2) `MutableList`는 맨 뒤에 추가(`add`)하는 게 O(1)인 반면 맨 앞에 추가(`add(0, ...)`)는 기존 원소를 전부 밀어야 해서 O(n). 그래서 저장은 등록 순서(가장 저렴한 방식)로 유지하고, 화면에 보여줄 때만 `.reversed()`로 변환하는 게 효율적. "저장 구조는 단순하게, 표시는 필요할 때 가공"하는 패턴.
