# 하네스 백로그

프로젝트가 진행되며 조건이 갖춰지면 추가할 하네스 항목들. 지금 당장은 조건이 안 맞아서(예: 테스트 코드가 아직 없음) 미룬 것들이므로, 아래 "트리거 시점"이 되면 이 문서를 참고해 나(에이전트)에게 요청하면 바로 적용 가능.

## 1. 커밋 전 빌드 게이트

- **트리거 시점**: 지금부터 (코드가 생기는 즉시)
- **하려는 것**: `git commit` 실행 전에 `gradlew build`가 성공해야만 커밋되도록 강제
- **적용 방법**: `.claude/settings.json`에 `PreToolUse` 훅 추가. `Bash` 도구 호출 중 명령어가 `git commit`으로 시작하면, 먼저 `gradlew build`를 실행해서 실패 시 훅이 커밋을 차단 (`continue: false`)
- **요청 예시**: "커밋 전에 빌드 검증하는 훅 추가해줘"

## 2. 테스트 게이트 (12단계 이후)

- **트리거 시점**: 12단계(TDD/JUnit 도입) 시작 이후
- **하려는 것**: 테스트 코드가 생긴 뒤부터는 커밋 전에 `gradlew test`도 통과해야 함
- **적용 방법**: 위 빌드 게이트 훅에 테스트 실행 단계 추가
- **요청 예시**: "이제 테스트 코드 생겼으니까 커밋 전에 테스트도 통과해야 하게 해줘"

## 3. 학습 로그 완결성 체크

- **트리거 시점**: 학습 로그 시스템(`docs/learning-log/`) 운영이 안정된 후, 원하면 지금도 가능
- **하려는 것**: 단계 커밋 시 해당 `step-NN.md` 파일이 함께 staged 되어 있는지 확인. 빠졌으면 경고
- **적용 방법**: `PreToolUse` 훅에서 `git commit` 명령을 가로채 `git diff --cached --name-only`로 스테이징된 파일 중 `docs/learning-log/step-*.md`가 있는지 검사
- **요청 예시**: "커밋할 때 학습로그 파일도 같이 올라가는지 체크하는 훅 추가해줘"

## 4. 커밋 메시지 형식 강제

- **트리거 시점**: 커밋 메시지가 형식에서 자꾸 벗어난다고 느껴질 때
- **하려는 것**: `N단계: <요약>` 형식이 아니면 커밋 거부
- **적용 방법**: 실제 git의 `commit-msg` 훅(Claude Code 훅이 아니라 `.git/hooks/commit-msg`) 사용 — 이건 사람이 직접 커밋해도 걸리는 진짜 git 레벨 강제
- **요청 예시**: "커밋 메시지 형식 강제하는 git hook 만들어줘"
